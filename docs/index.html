CoCreate-filter

CoCreate-filter javascript component. Easy configuration using data-attributes and highly styleable.

Install

 
npm install @cocreate/filter

Or you can use cdn link:

<script>https://cdn.cocreate.app/filter/latest/CoCreate-filter.min.js</script>

Dependencies components

CoCreate.js, CoCreate-utils.js

Usage

CoCreate-filter is basic component that make the the readDocumentList parameters to fetch the data from db by attributes.

Example

HTML Template


<div data-template_id="filter" data-filter_name="name" data-filter_value="testing" data-filter_operator="$eq"></div>
<div data-template_id="filter" data-filter_name="age" data-filter_value="10" data-filter_operator="$gt"></div>
<div data-template_id="filter" data-order_by="age" data-order_type="asc"></div>
...
<input data-template_id="filter" data-filter_name="description" data-filter_operator="$in"/>
....

<div data-fetch_collection="testing" data-template_id="filter"
        data-filter_name="filter"
        data-filter_value="testing"
        data-fitler_operator="$ne"
>....</div>

Attributes

  • data-{container_id} string required

    This attribute should be defined. This attribute name is difference for each component.

    Template wrapper case: the attribute name is data-template_id

  • data-filter_name string optional

    Collections's field name for filter.

  • data-filter_value string optional

    Collection's field value for filter. If data-filter_name defined, this attribute should defined too

  • data-filter_value_type string optional

    string | raw . Default value is string

    raw case, it will make values array from string by "," separation

  • data-filter_operator string optional

    Define the filter operator.

    Operators: $contain | $range | $eq | $ne | $lt | $lte | $gt | $gte | $in | $nin | $geoWithin

  • data-order_by string optional

    Field name to sorting.

  • value string optional

    The value for sorting.

    value="asc | desc". Default value is asc

  • data-fetch_type string optional

    Available value: loadmore | scroll | total.

  • data-export_type string optional

    To export the fetched data, This attribute will be use

    Available value: json | cvs

  • data-import string optional

    When import the file content into selected collection, This attribute will be use

    Available value: true

Functions

  • setFilter(elements, mainAttri, type)

    To apply filter into the componentis, the component should call this function

    - elements Type:Noderequired

    Dom elements for container

    - mainAttr Type:stringrequired

    This is the container's attribute name. Ex: template wrapper case- data-template_id

    - typeType:stringoptional

  • fetchData(item)

    To fetch the data, the component should call this function.

    - item Type:objectrequired

    This paramter is the returned value by setFilter() function

How Does It Works

CoCreate-filter build the filter object based on attributes

To make filter object , Filter should define data-{container_id}
